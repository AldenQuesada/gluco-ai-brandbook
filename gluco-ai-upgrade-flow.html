Parte 6 — Recomendações para Reduzir Custos
🧠
1. Haiku para tudo — só use Sonnet quando realmente necessário
Claude Haiku 3.5 cobre 95% dos casos do Gluco AI. Sonnet custa 3–4x mais. Use Sonnet apenas para relatórios mensais complexos ou quando o usuário fizer perguntas muito abertas sobre condição médica.
Economia: -70% no custo de tokens
💾
2. Prompt Caching — cachear o system prompt do usuário
O perfil do usuário (tipo de diabetes, HbA1c, medicação, complicações, histórico) é enviado em todo request. Com prompt caching do Claude, tokens repetidos custam 90% menos. Implementar cache_control: ephemeral no bloco de perfil.
cache_control: {{ type: "ephemeral" }} — no system prompt block
Economia: -40% no custo total de tokens
👁️
3. OCR dedicado para leitura de glicemia (não LLM)
Foto do glicosímetro → use Google Vision OCR ou Tesseract para extrair apenas o número. Custo: ~$0,0001/imagem (25x mais barato que LLM). Só passe ao LLM o número extraído + contexto. O LLM não precisa ver a imagem.
glycemia_value = ocr.extract(image) → llm.analyze(value, user_profile)
Economia: -90% no custo das leituras de glicemia
📝
4. Templates para o ritual diário do Plano 2
As 9 mensagens diárias de ritual (jejum, café, lanche…) do Plano 2 NÃO precisam ser geradas por LLM toda vez. Criar 30 variações por momento do dia + substituição de variáveis ({nome}, {meta_glicemia}, {medicacao}). LLM só quando o usuário responde com uma pergunta nova.
ritual_msg = template_engine.render(slot, user_vars) — sem LLM
Economia: -80% no custo do Plano 2
🍽️
5. Cache de alimentos comuns
70% das fotos de comida no Brasil são arroz, feijão, frango, macarrão, pão. Criar um cache de resposta para os 50 alimentos mais comuns. Se a foto é identificada como um desses, retornar resposta cacheada + personalizar apenas os números de impacto glicêmico pelo perfil.
food_cache.get(food_hash) ?? llm.analyze(image)
Economia: -55% no custo de análise de comida
📦
6. Comprimir histórico antes de enviar ao LLM
Não enviar o histórico completo de mensagens a cada request. Manter apenas os últimos 3 dias em contexto + um resumo comprimido dos últimos 30 dias. Implementar janela deslizante de contexto.
context = last_3_days(messages) + summarize(30_days_history)
Economia: -35% nos tokens de contexto

7. Decisão técnica — qual modelo usar onde
Foto de Glicemia → GPT-4o mini
Tarefa simples: apenas ler um número no display. GPT-4o mini resolve com 99%+ de acurácia. Custo: R$0,0005/foto. Não há motivo para usar modelo mais caro aqui.
model: "gpt-4o-mini" · detail: "low"
Foto de Alimento → Claude Haiku ou GPT-4o mini
Tarefa mais complexa: identificar alimentos, estimar porções, calcular índice glicêmico. Claude Haiku entrega melhor resultado. Se orçamento for crítico, GPT-4o mini também funciona bem. Custo Haiku: R$0,018/foto.
model: "claude-haiku-4-5" · detail: "high"
